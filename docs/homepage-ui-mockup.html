图片系统重构

这是当前最大的短板。方案从「凑合用图」升级为「专业图片管道」。核心变化：不再用 Steam CDN 直链，改为自建图片服务。

1

统一图片服务

通过 Cloudflare Worker 代理图片，在边缘做格式转换（WebP/AVIF）、尺寸裁剪、质量压缩。支持 ?w=300&q=80 参数控制。

2

多尺寸 srcSet

每个图片生成 3 个尺寸：small (300w)、medium (600w)、large (900w)。浏览器根据视口选择合适的尺寸加载。

3

模糊占位 + 渐变过渡

从 DOM 读取 dominant_color 作为背景色，图片加载时用 blur(20px) 过渡到清晰图。用户几乎感知不到加载过程。

4

优雅降级

不支持 WebP 的旧浏览器自动 fallback 到 JPEG。图片加载失败时显示渐变色占位 + 游戏名称，不显示裂图。
